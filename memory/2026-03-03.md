## Lessons Learned
- Always check `tailscale serve` compatibility before recommending it — macOS only in practice
- Check your own working config BEFORE suggesting changes to peer agents
- Next.js production API routes cache aggressively — always add `dynamic = 'force-dynamic'`
- Agent pollers need explicit message type filters or they'll chat-loop forever
- `openclaw config set` (not `openclaw gateway config set`) — correct syntax confirmed on Windows too
- **Azure portal custom checkboxes**: `element.focus()` + Space. Class: `fxc-gc-row`. JS `.click()` blocked.
- **Azure app registration flow**: Register → API permissions → Microsoft Graph → Delegated → search → focus row → Space → Add permissions → Grant admin consent → Certificates & secrets
- **node:sqlite** = built-in Node 22.5+, zero compilation, cross-platform — always prefer over better-sqlite3 for portability
- **Convex free tier = 2 deployments max** — one project with dev+prod counts as 2. Plan accordingly.
- **Peer agent workspaces need backup** — identity wipe without backup = start from scratch
- **Agent credentials should never travel in plaintext over the bus** — use a secrets manager or manual paste
