## Key Decisions

- **`tailscale serve` crashes on Windows** — never suggest it for Dubya again. `bind: lan` is the Windows-compatible solution
- **`openclaw gateway restart` blocked on Cornerstone** — workaround is `launchctl unload/load` on the plist
- **`openclaw config set gateway.bind tailnet`** takes the agent off local — don't do it. `lan` keeps both local AND Tailscale
- **Pollers run on B.O.B.'s gateway, not Dubya/Cornerstone's own gateways** — they send heartbeats/responses AS those agents (agentId hardcoded in payload), but the AI doing the work is B.O.B.'s model. Good enough for now, not true autonomy
- **Chat loop root cause**: pollers had no message type filter — responded to everything including greetings and acknowledgments. Fixed with `messageType == 'request'` gate
- **`export const dynamic = 'force-dynamic'`** required in Next.js API routes to prevent stale cached responses in production
