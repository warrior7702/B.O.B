## Root Cause Found: Dubya Has Zero Tools (8:22 AM)
- Reviewed Dubya's `openclaw.json` — no `tools` section at all
- Without a `tools` block, OpenClaw gives the agent NO tool access — pure chatbot
- This is why he kept saying "I can't write files" — he literally couldn't. Not lying, not broken model.
- Fix: add `"tools": { "profile": "full", "exec": { "security": "full" } }` to top-level config
- Also locked model to `openrouter/deepseek/deepseek-r1` explicitly (was `openrouter/auto` — lets OpenRouter pick randomly)
- Fixed config saved to: `~/.openclaw/workspace/dubya-openclaw.json`
- **Lesson: ALWAYS check tools config before diagnosing model/persona problems**
- **Lesson: Missing `tools` block = chatbot mode. This is the #1 thing to verify on any new agent.**
- Deploy steps: stop gateway → overwrite openclaw.json → clear sessions → start gateway → fresh chat
